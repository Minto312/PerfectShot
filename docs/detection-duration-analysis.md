# 目開き判定に必要な秒数の理論的導出

## 背景

2006年イグノーベル賞（数学賞）を受賞した Nic Svenson & Piers Barnes の研究
"Blink-Free Photos, Guaranteed" (Velocity, CSIRO, June 2006) に基づき、
PerfectShot アプリの連続フレーム判定に必要な秒数を導出する。

### 元研究の概要

集合写真で全員の目が開いている1枚を得るために必要な撮影枚数を、
瞬きの確率モデルから算出した研究。

- 人間は平均 **10回/分** 瞬きし、1回の瞬きは約 **250ms**
- 各人の瞬きは **独立事象**（他人の瞬きに影響されない）
- 1人が任意の瞬間に目を開けている確率: **1 - λd ≈ 95.8%**
- n人全員が同時に目を開けている確率: **(1 - λd)^n**

この確率は有限人数では必ず 0 より大きいため、
連続的な時間の中で「全員が目を開けている瞬間」は確率1で存在する。

## パラメータ（Svenson-Barnes 準拠）

| パラメータ | 記号 | 値 | 説明 |
|---|---|---|---|
| 瞬き頻度 | λ | 10/60 = 1/6 回/秒 | 1人あたり |
| 瞬き持続時間 | d | 0.25 秒 | 1回あたり |
| 目を開けている確率（1人） | p₁ | 1 - λd = 23/24 ≈ 0.9583 | 任意の瞬間 |

---

## 導出

### Step 1: n人全員が目を開けている瞬間確率

各人の瞬きは独立なので:

```
p(n) = p₁ⁿ = (23/24)ⁿ ≈ 0.9583ⁿ
```

### Step 2: 連続フレーム判定への適用

PerfectShot は動画フレームを連続的に取得・判定する。
ただし、1回の瞬きが約 250ms (= 30fps で約7-8フレーム) 続くため、
連続フレーム間には相関がある。

**有効独立サンプルレート:**

```
r_eff = 1/d = 1/0.25 = 4 回/秒
```

瞬き持続時間 d 以上離れたフレーム同士は近似的に独立とみなせる。

### Step 3: 必要な判定秒数

99% 信頼度で少なくとも1枚の「全員目開き」フレームを得るために、
必要な独立サンプル数 k:

```
P(k回すべて失敗) = (1 - p(n))^k ≤ 0.01

k = ⌈ ln(0.01) / ln(1 - p(n)) ⌉
```

必要秒数:

```
T(n) = k / r_eff = k / 4
```

### Step 4: 安全係数

以下の現実的要因を考慮し、安全係数 α = 2 を適用:

- 瞬き頻度の個人差（5〜20回/分）
- 瞬き持続時間の個人差（100〜400ms）
- 環境要因（明るさ、緊張）によるばらつき
- 独立性仮定の誤差（瞬きの社会的伝播など）

```
T_safe(n) = α × k / r_eff = 2 × k / 4
```

## 実装提案

### 関数

```typescript
/**
 * 検出された人数に基づき、目開き判定に必要な秒数を算出する。
 * Svenson-Barnes (2006) の瞬きモデルに基づく。
 *
 * @param n - 検出された人数
 * @param confidence - 信頼度 (デフォルト: 0.99 = 99%)
 * @param safetyFactor - 安全係数 (デフォルト: 2)
 * @returns 推奨判定秒数
 */
function getRequiredSeconds(
  n: number,
  confidence = 0.99,
  safetyFactor = 2,
): number {
  const BLINK_RATE = 10 / 60;                       // λ: 瞬き/秒
  const BLINK_DURATION = 0.25;                       // d: 秒
  const EFFECTIVE_SAMPLE_RATE = 1 / BLINK_DURATION;  // r_eff: 4/秒

  const pAllOpen = Math.pow(1 - BLINK_RATE * BLINK_DURATION, n);
  const k = Math.ceil(
    Math.log(1 - confidence) / Math.log(1 - pAllOpen),
  );
  const T = (safetyFactor * k) / EFFECTIVE_SAMPLE_RATE;

  return Math.max(3, Math.min(T, 15)); // 最低3秒、最大15秒
}
```

### 設計方針

| 項目 | 値 | 理由 |
|---|---|---|
| 下限 | 3秒 | 少人数でも安定したフレーム選択のため |
| 上限 | 15秒 | UX上、長すぎる待機はユーザー離脱を招く |
| 安全係数 | 2 | 個人差・環境差のバッファ |
| 信頼度 | 99% | 撮り直しコストを最小化 |

### 現行タイマーとの統合

現在のタイマー（0/3/5/10秒カウントダウン）とは独立に、
カウントダウン完了後の「判定ウィンドウ」として動的に適用する:

```
[カウントダウン] → [判定ウィンドウ: T(n)秒] → [ベストフレーム選択] → [プレビュー]
```

## まとめ

- Svenson-Barnes の瞬きモデルにより、n人の集合写真で全員が目を開けている
  瞬間は連続観測すれば**確率1で存在する**ことが保証される
- 99%信頼度 + 安全係数2で、1〜10人は**3秒**、20人で**約5秒**、30人で**約8秒**
- アプリ実装では `getRequiredSeconds(n)` で人数に応じた動的判定時間を算出し、
  固定タイマーに代わる合理的なUXを提供できる
